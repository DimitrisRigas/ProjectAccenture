Project overview

AI Knowledge Engineering for regulatory compliance.

This project implements an end-to-end AI-powered regulatory compliance assistant for EU financial institutions. It combines governed data engineering, Gold embedding generation, vector search, hybrid retrieval, source-grounded generation, PII handling, Dockerized application execution, and automated RAG evaluation.

Business challenge

EU financial institutions must continuously interpret complex regulations such as GDPR, DORA, PSD2, MiFID II, and the EU AI Act. Compliance teams need fast, contextual, and traceable access to regulatory obligations, controls, and requirements.

Solution objective

The assistant enables users to ask natural-language compliance questions and receive grounded answers supported by retrieved regulatory sources, metadata, page numbers, file names, and official source URLs.

Implemented architecture

The solution uses Databricks for governed document processing, Delta Lake for Bronze/Silver/Gold tables, Azure OpenAI for Gold embedding generation and GPT-4o answer generation, Azure AI Search for hybrid retrieval, Docker for local application execution, and FastAPI for the API layer.

1
Regulatory document acquisition PDF, HTML, and XML regulatory documents are collected or registered from official EU sources.
2
Metadata manifest A manifest tracks regulation title, short title, CELEX identifier, source URL, file format, and lineage information.
3
Databricks Volume Raw files and metadata are uploaded into a Unity Catalog Volume for governed cloud storage.
4
Bronze ingestion Databricks extracts text from PDFs, HTML, and XML files and stores document-level and page-level records.
5
Silver chunking Extracted text is split into overlapping AI-ready chunks while preserving metadata and source provenance.
6
Gold embedding generation The Gold job generates embeddings using Azure OpenAI and writes chunk text, metadata, and vectors to the Gold Delta table.
7
Azure AI Search upload The upload job reads existing Gold embeddings and uploads chunk text, metadata, and vectors to Azure AI Search.
8
RAG API and frontend FastAPI exposes the /ask endpoint and this frontend displays grounded answers with retrieved evidence.

Implemented capabilities

Governed ingestion Raw regulatory documents are stored and processed through Databricks and Delta tables.
AI-ready chunking Long regulatory documents are converted into structured chunks with metadata and lineage.
Gold embeddings Azure OpenAI embeddings are generated in the Gold layer and reused during Azure AI Search upload.
Hybrid retrieval Azure AI Search combines keyword and vector retrieval for more reliable regulatory search.
Source-grounded answers GPT-4o generates answers using retrieved source context and source labels.
PII redaction User questions are checked for emails, phones, IBAN-like values, and card-like numbers.
Dockerized API The local FastAPI application layer runs through Docker Compose for reproducible demos.
Automated evaluation The RAG pipeline is evaluated using retrieval checks and LLM-as-judge groundedness scoring.
Source traceability Returned answers include regulation title, file name, source URL, page number, and retrieval score.

Dockerized application layer

The local FastAPI backend and RAG service are Dockerized using Docker Compose. Databricks, Azure OpenAI, and Azure AI Search remain external managed cloud services. Runtime configuration is injected through the `.env` file and is not copied into the image.
