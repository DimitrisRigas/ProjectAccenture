Project overview

Built for governed regulatory intelligence.

This project implements an AI-powered compliance assistant for EU financial institutions. It combines regulatory document ingestion, Databricks data engineering, Delta tables, Azure OpenAI embeddings, Azure AI Search, FastAPI, and retrieval-augmented generation into one end-to-end knowledge platform.

Business problem

Financial institutions must continuously understand and comply with complex EU regulations such as GDPR, DORA, PSD2, MiFID II, and the EU AI Act. Traditional keyword search is often not enough because compliance questions require context, semantic retrieval, and traceable answers.

Project objective

The goal is to provide a regulatory compliance assistant that accepts natural-language questions and returns grounded answers based on retrieved regulatory chunks, with clear references to the source regulation, file, page number, and URL.

Implemented architecture

The system uses Databricks as the governed data engineering layer and Azure AI Search as the application retrieval layer. Raw regulatory files are uploaded to Databricks, transformed into Bronze/Silver/Gold Delta tables, embedded through Azure OpenAI, indexed in Azure AI Search, and queried by a FastAPI backend connected to this frontend.

1
Document acquisition Regulatory documents are collected from EUR-Lex and other official EU sources in PDF, HTML, and XML formats.
2
Metadata manifest A manifest tracks CELEX ID, short title, regulation title, source URL, file format, SHA256 hash, and ingestion status.
3
Databricks Volume Raw regulatory files and metadata are uploaded to a Unity Catalog Volume for governed storage.
4
Bronze tables The Bronze job extracts raw document text and stores document-level and page-level regulatory data in Delta tables.
5
Silver chunks The Silver job splits extracted text into overlapping, AI-ready chunks while preserving metadata and lineage.
6
Gold retrieval table The Gold job prepares clean chunks and metadata such as short title, regulation title, source URL, page number, and file name.
7
Azure AI Search index The pipeline creates an Azure AI Search vector index and uploads embedded chunks using Azure OpenAI text-embedding-3-small.
8
FastAPI RAG service The backend exposes a /ask endpoint that embeds the question, retrieves relevant sources, and generates an answer using GPT-4o.
9
Frontend interface The web interface lets users ask compliance questions and inspect the retrieved evidence behind each answer.

Current status

The end-to-end pipeline has been completed. The Databricks Bundle runs Bronze ingestion, Silver chunking, Gold preparation, Azure AI Search index creation, Azure OpenAI embedding generation, and document upload. The FastAPI backend successfully answers questions through the /ask endpoint.

Technologies used

The implementation combines Python, uv, FastAPI, Databricks, Unity Catalog, Delta Lake, Databricks Bundles, PySpark, Azure OpenAI, Azure AI Search, GPT-4o, and a modern HTML/CSS/JavaScript frontend.

Python FastAPI Databricks Delta Lake Unity Catalog PySpark Azure OpenAI Azure AI Search GPT-4o RAG

Component responsibilities

The solution is split into three main layers. Databricks handles governed ingestion and transformation, Azure handles embeddings and vector search, and the FastAPI/frontend layer exposes the final assistant to users.

Data engineering Downloader, uploader, manifest, Databricks Volume, Bronze/Silver/Gold Delta tables.
AI retrieval Azure OpenAI embeddings, Azure AI Search indexing, semantic retrieval, and GPT-4o generation.
Application layer FastAPI /ask endpoint, frontend interface, source rendering, and final RAG integration.
